# 文章扫描脚本使用说明

## 脚本文件
- `scan_articles_direct.py` - 直接数据库版本的文章扫描脚本
- `scan_articles_direct.bat` - Windows批处理文件，方便运行脚本

## 功能特点

### 直接数据库操作
- 直接将文章数据写入Django数据库，无需通过API
- 避免了API认证和网络请求的复杂性
- 处理速度更快，更稳定

### 智能文章处理
- 自动扫描 `generated_articles` 目录下的所有文章
- 支持多种分类：汽车、日记、教育、娱乐、美食、生活、民生、情书、旅游、冷知识、职场
- 自动解析文章元数据（标题、摘要、内容等）
- 自动生成文章别名（slug）
- 自动复制文章文件到静态目录

### 分类管理
- 自动创建不存在的分类
- 支持分类缓存，提高处理效率
- 分类映射：英文目录名 → 中文分类名

### 用户管理
- 自动创建默认作者（anonymous_user）
- 支持指定文章作者

### 安全特性
- 支持试运行模式（--dry-run），不实际写入数据库
- 支持处理数量限制（--limit），避免一次性处理过多文章
- 详细的日志记录，便于调试和监控

## 使用方法

### 基本用法
```bash
# 处理所有文章
python scan_articles_direct.py

# 使用批处理文件
scan_articles_direct.bat
```

### 高级用法
```bash
# 试运行模式，不实际写入数据库
python scan_articles_direct.py --dry-run

# 限制处理数量
python scan_articles_direct.py --limit 10

# 试运行 + 限制数量
python scan_articles_direct.py --dry-run --limit 5
```

## 输出信息

### 实时日志
- 分类扫描进度
- 文章处理进度
- 文件复制状态
- 数据库写入结果

### 统计报告
- 发现文章总数
- 处理文章总数
- 成功/失败数量
- 各分类文章数量
- 处理成功率

### 日志文件
- 详细日志保存在 `scan_articles_direct.log`
- 包含所有处理细节和错误信息

## 注意事项

1. **数据库连接**：确保Django项目的数据库配置正确
2. **文件权限**：确保脚本有权限读取源文件和写入目标目录
3. **重复处理**：脚本会跳过已存在的文章（基于别名判断）
4. **内存使用**：处理大量文章时注意内存使用情况

## 故障排除

### 常见问题
1. **Django配置错误**：检查 `DJANGO_SETTINGS_MODULE` 环境变量
2. **数据库连接失败**：检查数据库配置和权限
3. **文件路径错误**：确保 `generated_articles` 目录存在
4. **权限不足**：确保有读写文件的权限

### 调试建议
1. 使用 `--dry-run` 模式测试
2. 使用 `--limit 1` 处理单篇文章测试
3. 查看详细日志文件
4. 检查数据库中的数据

## 性能优化

- 使用分类缓存减少数据库查询
- 批量处理文件操作
- 合理设置处理数量限制
- 定期清理日志文件
